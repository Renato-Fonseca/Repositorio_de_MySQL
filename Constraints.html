Comandos Constraints Tipos Primitivos Outras Coisas

Antes de tudo, o que são os sinais de # do repositório?

Para exemplificar a sintaxe de comandos e constraints no repositório, eu vou usar sinais de # para indicar nomes ou palavras váriaveis

por exemplo, para explicar o comando create database eu vou mostrar isso: create database #nome do banco de dados#;, com esse exemplo de sintaxe, eu quero dizer que você vai usar "create database", daí o nome do banco de dados que você escolher, daí ponto e vírgula, eu não quero dizer para você escrever exatamente "create database #nome do banco de dados#;", e sim para substituir os sinais de # e o que está dentro pelo nome do banco de dados de sua escolha, deu para entender

Criação de bancos de dados e tabelas

Padrão de caracteres

default character set utf8mb4: permite o uso de caracteres latinos no banco, sempre use essa configuração quando você for criar um banco de dados

default collate utf8mb4_general_ci: faz a mesma coisa que o default character set utf8mb4, sempre use essas duas constraints quando estiver criando seu banco de dados

default charset = utf8mb4: permite o uso de caracteres latinos na tabela, sempre use isso quando for criar suas tabelas se elas tiverem palavras em português por exemplo, para usar ele você precisa coloca-lo depois do fechamento dos parentêses que contém os campos

Condições necessárias para a criação

if not exists: essa constraint é muito útil e você pode usar em muita coisa, por exemplo, se você usar if not exists quando você estiver criando um banco ou uma tabela, ele só vai criar o banco ou tabela se ele(a) ainda não existir, acho que ficou bem claro. Para usar ele na criação de uma tabela por exemplo, você precisa inserir ele no meio do table e do nome da tabela. Digamos que o nome é "minha_tabela", então você iria usar create table if not exists minha_tabela(campos);, desta forma ele só iria criar a tabela "minha_tabela" se ela ainda não existisse, você usa a mesma sintaxe com o banco

if exists: essa constraint também é muito útil e dá para usar em vários casos, ela é o contrario do if not exists, quando você coloca ela quando estiver deletando um banco de dados ou tabela, ele(a) só vai ser deletado(a) se ele(a) existir. Para usar essa constraint você usa a mesma sintaxe que a anterior, digamos que queira deletar o banco de dados "meu_banco" só se ele existir, então você colocaria o if exists no meio do database e do nome do banco, ficaria drop database if exists meu_banco;

Criação ou alteração de campos(colunas) de tabelas

not null: impede que o campo seja nulo, ou seja, esse campo precisará ser preenchido com algo, mesmo que seja um espaço

Definindo o padrão caso nada seja inserido

default: define o valor padrão do campo, ou seja, se não for inserido nada, o campo receberá o valor padrão, para usar ele é só dar um espaço à direita dele e colocar o valor padrão entre aspas, ficando default '#exemplo de valor padrão#'

auto_increment: essa constraint é usada em campos de valores numéricos, ela faz uma contagem de um em um que vai ser usada no campo caso não seja inserido nada nele, todos os campos auto_increment devem ser uma chave. Por exemplo, digamos que você tenha o campo id, esse campo você definiu como int, not null, auto_increment e como uma primary key, se você não informar o id no primeiro registro, o id vai receber valor 1, no segundo registro valor 2, no terceiro valor 3, e assim vai, lembrando que isso só acontece se você não informar o id no caso, se você inserir o número 10 no id do quarto registro, o id do quinto registro vai ser 11, o sexto 12, deu pra entender

Definindo os campos chave

primery key: define um campo como chave primária, ou seja, esse campo vai ser único, não vão ter dois registros com o mesmo valor nesse campo, e esse campo vai ser usado para identificar cada registro. Para usar ele, você coloca ele em outro espaço do parentêses com os campo da tabela, como se fosse um campo diferente, com um parentêses à direita dele com o campo que você quer definir como chave primária. Digamos que o campo que você quer colocar como chave primária seja o campo 'id', então você daria uma vírgula e colocaria primary key (id), como se fosse outro campo, e então o campo id iria virar uma chave primária

Tornando um campo único

unique: define um campo como único, campos únicos não podem ter dois registros com o mesmo valor. Se o campo "nome" for único por exemplo, não vão ter dois nomes iguais. Não confunda unique com primary key, um campo primary key é único, mas eles também é outras coisas, um campo unique não serve para identificar um registro

Proibindo números negativos

unsigned: em campos do tipo numérico, você pode proibir que sejam inseridos valores negativos usando a constraint unsigned,dessa forma o máximo de números positivos que podem ser inseridos é dobrado
