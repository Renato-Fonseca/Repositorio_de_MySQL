Entendendo banco de dados e tabelas

O MySQL é um sistema de gerenciamento de banco de dados (SGBD), que utiliza a linguagem SQL

Um banco de dados é um local onde você pode armazenar dados , todo jogo tem um banco de dados por exemplo

Em um banco de dados existem tabelas que guardam os dados, é isso mesmo que você está pensando, uma tabela com linhas e colunas, uma tabela organiza os dados em campos, que são as colunas da tabela. Em um jogo por exemplo, toda característica de sua conta ocupa uma coluna de uma certa linha de uma tabela(todas na mesma linha, cada uma em uma coluna diferente, deu para entender).

Cada linha da tabela(linha horizontal) completa com dados é chamada de registro

mas é claro que existem vários tipos de tabela em um jogo, eu só exemplifiquei com uma tabela que guarda as características de sua conta, mas existem muitas outras

Classificação dos Comandos

a maioria dos comandos de MySQL tem classificações dependendo de sua função

as duas classificações mais comuns são DDl e DML

DDL significa "Database Definition Language", comandos DDL são comandos que interferem na ESTRUTURA de um banco de dados ou tabela, os campos de uma tabela são um exemplo de estrutura de uma tabela

DML significa "Database Manipulation Language", comandos DML são comandos que interferem nos DADOS de um banco de dados ou tabela, não na estrutura

Antes de tudo, o que são os sinais de # do repositório?

Para exemplificar a sintaxe de comandos e constraints no repositório, eu vou usar sinais de # para indicar nomes ou palavras váriaveis

por exemplo, para explicar o comando create database eu vou mostrar isso: create database #nome do banco de dados#;, com esse exemplo de sintaxe, eu quero dizer que você vai usar "create database", daí o nome do banco de dados que você escolher, daí ponto e vírgula, eu não quero dizer para você escrever exatamente "create database #nome do banco de dados#;", e sim para substituir os sinais de # e o que está dentro pelo nome do banco de dados de sua escolha, deu para entender

Criando banco de dados e tabelas

create database

você usa o comando create database, de classificação DDL(por criar a ESTRUTURA do banco), para criar um banco de dados, a sintaxe é assim:

create database #nome do banco de dados#;

mas na maioria das vezes você vai colocar as constraints "default charset set utf8mb4" e "default collate utf8mb4_general_ci" para informar que o banco tem palavras latinas(como palavras brasileiras) e permitir caracteres acentuados no seu banco, para isso você vai usar a seguinte sintaxe:

create database #nome do banco de dados#
default character set utf8mb4
default collate utf8mb4_general_ci;

sempre que seu banco de dados tiver palavras em português ou espanhol use essa sintaxe mais completa, isso evita muitos problemas

você pode criar um banco apenas se ele ainda não existir com o if not exists, adicionando com a sintaxe anterior fica:

create database if not exists #nome do banco de dados#
default character set utf8mb4
default collate utf8mb4_general_ci;

create table

para criar uma tabela em um banco de dados, você usa o comando create table de classificação DDL, aqui está a sintaxe:

create table #nome da tabela#(#campos, seus tipos e suas constraints#);

Ao criar uma tabela, você precisa criar seus campos dentro dos parentêses depois do nome da tabela, ao criar um campo, você precisa informar o nome dele, o tipo e as constraints(as constraints são opcionais), nessa ordem, e lembrando que você tem que separar cada campo por vírgulas, preste atenção na sintaxe de exemplo para entender

digamos que queira criar uma tabela chamada 'minha_tabela', com os campos nome, que é do tipo varchar(20) e com a constraint 'not null', altura, que é do tipo decimal(3, 2), e genero, que é do tipo enum('M', 'F'), caso não conheça esses tipos primitivos, é só dar uma olhada na página de tipos primitivos do repositório

dados os nomes dos campos e os tipos deles, para criar a tabela 'minha_tabela' com esses campos você faria assim:

create table minha_tabela(
nome varchar(20) not null,
altura decimal(3, 2),
genero enum('M', 'F')
);

você pode usar a constraint default charset utf8mb4 no final do comando, isso vai permitir caracteres acentuados e caracteres latinos na tabela, ficaria assim:

create table minha_tabela(
nome varchar(20) not null
altura decimal(3, 2)
genero enum('M', 'F')
) default charset = utf8mb4;

sempre use isso quando sua tabela tiver palavras em português ou espanhol

você pode criar uma tabela apenas se ela ainda não existir com o if not exists, adicionando com a sintaxe anterior fica:

create table if not exists minha_tabela(
nome varchar(20) not null
altura decimal(3, 2)
genero enum('M', 'F')
) default charset = utf8mb4;

Inserindo Dados em Tabelas

Agora que você já aprendeu a montar a estrutura de uma tabela, vamos colocar dados nela

Para isso, usamos o insert into, da classificação DML

informando a ordem que os valores serão passados(jeito completo)

ele tem vários tipos de sintaxe, a sintaxe padrão dela é:

insert into #nome da tabela#(#campos#) values(#valores#)

você deve informar os campos separados por vírgulas no primeiro parênteses, e no segundo os valores, cada valor dentro de aspas e separados entre si por vírgulas

lembre-se, tudo que é dado no MySQL deve estar dentro de aspas

nessa sintaxe você informa no primeiro parênteses a ordem dos campos e no segundo você coloca os dados NA ORDEM dos campos inseridos no primeiro parênteses

nesse caso, os valores que você informou no segundo parentêses vão ser colocados respectivamente nos campos dos primeiros, ou seja, nessa sintaxe você pode escolher a ordem que os valores vão ser passados

usando a ordem dos campos da tabela(jeito simplificado)

se você for passar os valores na mesma ordem que os campos estão organizados na tabela você não precisa usar os primeiros parênteses, a sintaxe ficaria assim:

insert into #nome da tabela# values(#valores#)

vários registros de uma vez

invez de usar um insert into para cada registro que você fizer, você pode fazer todos os registros em um mesmo insert into, economizando muito tempo

Bom, nas sintaxes anteriores você pode ver que tem apenas um parentêses com os dados depois do values, você pode colocar mais se você quiser, daí você separa cada parentêses com vírgulas entre eles, dessa forma você pode fazer quantos registros você quiser de uma vez só, isso funciona tanto com o jeito completo como o simplificado

lembre-se de colocar ; só depois do último parentêses

Usando constraints

invés de dar um valor para um certo campo, você também pode usar uma constraint

Por exemplo, digamos que você tenha o campo "id", ele é do tipo int e tem a constraint auto_increment, nesse caso se você não iria querer passar um número manualmente para esse campo porque ele é auto_increment, então ao invés de você colocar um número, você coloca default, daí o id receberia um valor com base no auto_increment, porque é o valor padrão(default) dele

outro exemplo, digamos que você tenha o campo "nacionalidade", ele é do tipo varchar(20) e tem a constraint default"Brasil", então se você quisesse que esse campo tivesse o valor padrão, você também poderia usar o default

IMPORTANTE: quando você usa constraints no insert, como o default, você não pode colocar ele dentro de aspas se não dá errado, use ele sem aspas

Selecionando um banco de dados

quando você insere, muda, adiciona ou excluí informações em um banco de dados, você precisa selecionar ele, para isso você pode apertar duas vezes nele na parte de 'schemas', ou você pode usar o comando use, é só usar a sintaxe:

use #nome do banco de dados#;

quando você rodar esse comando, ele vai selecionar o banco de dados, daí depois você pode trabalhar com ele

lembrando uma coisa, digamos que você queira criar uma tabela em um banco, você usaria o comando use com esse banco SOZINHO, daí sim você pode criar a tabela

Alterando a Tabela

alter table

Esse é o comando que usamos para alterar tabelas, ele é da classificação DDL

dá pra usar ele de várias formas

add column

o add column adiciona uma coluna nova(um campo novo) à tabela

a sintaxe:

alter table #nome da tabela#
add column #nome da coluna nova# #tipo e constraints, respectivamente#;

se você usar essa sintaxe, o campo vai ser adicionado ao final da tabela

Você pode usar o after para posicionar essa coluna nova DEPOIS de alguma coluna existente na tabela

a sintaxe fica assim:

alter table #nome da tabela#
add column #nome da coluna nova# #tipo e constraints, respectivamente# after #coluna já existente na tabela#;

Se você quiser posicionar o campo novo no Inicio da tabela, você usa o first

olhe a sintaxe:

alter table #nome da tabela#
add column #nome da coluna# #tipo e constraints, respectivamente# first;

drop column

o drop column deleta uma coluna

veja a sintaxe:

alter table #nome da tabela#
drop column #nome da coluna#;

modify column

o modify column serve para mudar o tipo e as constraints de um campo, mas não dá pra mudar o nome, para mudar o nome você tem que usar o comando change column que já vamos ver

olhe a sintaxe:

alter table #nome da tabela#
modify column #nome da coluna# #tipos e constraints novos#;

e claro, você não pode deixar omitir o tipo se você quiser mudar só as constraints

por exemplo, digamos que você tenha a tabela pessoas, dentro dela você tem o campo nome, que é um varchar(20), e você quer adicionar a constraint not null, para isso você usaria "alter table pessoas modify column nome varchar(20) not null;" e não apenas "alter table pessoas modify column nome not null;", e aliás isso daria errado por causa do not null, mais pra baixo é falado sobre o problema e como resolver, confira!

change column

O change serve para mudar nome, tipo e constraints de um campo, ele não é muito útil quando você quer mudar só o nome ou só o tipo e/ou constraints porque dá muito trabalho, mas se seu objetivo é mudar tanto o nome quanto tipo e/ou constraints o change é pra você

a sintaxe dele:

alter table #nome da tabela#
change #nome antigo da coluna# #nome, tipo e contraint novos, respectivamente#;

Lembrando que você não pode omitir nada na parte de nome, tipo e constraints novos, o jeito que você colocar ali é como o campo vai ser, aqui um exemplo

Um erro comum e como consertar

Se você for criar ou modificar uma coluna e essa coluna tiver a nova constraint "not null" dará erro na hora da criação ou modificação porque os registros anteriores teriam o valor nulo nesse campo, e o campo não aceita valores nulos, veja uma aula do Guanabara sobre

rename to

Se você não quiser mudar o nome de um campo em específico mas sim o nome da tabela inteira você usa o rename to, a sintaxe:

alter table #nome antigo da tabela#
rename to #nome novo da tabela#;

add primary key()

Você pode adicionar uma chave primária para a tabela caso você tenha esquecido disso na criação da tabela

para isso você usa o add primary key()

a sintaxe:

alter table #nome da tabela#
add primary key(#nome do campo que você quer como chave primária#);

Alterando os dados da tabela

update

o update é um DML e serve para mudar colunas de registros que já estão dentro da tabela

Para usar ele, você primeiro usa o set para indicar qual vai ser a mudança, e depois um where para indicar em qual linha vai acontecer essa mudança

no set você coloca o campo que vai ser mudado = o novo valor entre aspas

no where você coloca um campo que vai ser levado em consideração para definir em qual linha vai ser a mudança, veja só a sintaxe:

update #nome da tabela#
set #nome do campo a ser mudado# = '#novo valor do campo#'
where #nome do campo levado em consideração# = "valor do campo levado em consideração";

vamos fazer um exemplo, digamos que você tenha a tabela "pessoas", nela tem o campo "id" que é uma chave primária, na tabela pessoas tem vários registros, e você quer mudar o campo "nome" de um dos registros para "Gustavo", esse registro tem o id 15

então para fazer essa mudança você usaria a sintaxe:

uptade pessoas
set nome = 'Gustavo'
where id = '15';

Você pode mudar vários campos de um registro de uma vez só, mas as colunas do mesmo registro

para isso, você coloca todos os campos a serem mudados com seus novos valores, exatamente do jeito que você já aprendeu, mas você separa eles por vírgulas

mais um exemplo, você tem a mesma tabela pessoas, com vários registros, e você quer mudar o campo "nome" para "Mateus" e "nacionalidade" para "Brasil", isso no mesmo update, e o registro que você quer mudar tem o id 24, então você usaria:

update pessoas
set nome = 'Mateus', nacionalidade = 'Brasil'
where id = '24';

para evitar que você digite o campo errado no where e mais de um regstro seja alterado(sim, isso é possível e muito perigso), você pode usar limit 1 no final do comando, sempre use o limit com o número de linhas que você deseja mudar

olha o exemplo anterior com o limit:

update pessoas
set nome = 'Mateus', nacionalidade = 'Brasil'
where id = '24'
limit 1;

delete from

o delete from serve para deletar um registro inteiro(ou vários), ele é da classificação DML

para usar ele você escolhe a tabela e depois usa um where para determinar qual(is) registros serão deletados, igual você faz com o update, ou seja, colocando um campo que será levado em consideração, olhe a sintaxe:

delete from #nome da tabela#
where #nome do campo levado em consideração# = '#valor campo levado em consideração#';

digamos que na tabela pessoas você queira eliminar os registros com o campo "ano" igual a 2060, então você usaria a sintaxe:

delete from pessoas
where ano = '2060';

no delete from você também pode usar o limit para ter mais segurança, olhe a mesma sintaxe com o limit:

delete from pessoas
where ano = '2060'
limit 1;

truncate

o truncate delete TODOS os dados de uma tabela, ele é da classificação DML é um comando incrivelmente perigoso, o truncate deleta os dados permanentemente

olha a sintaxe:

truncate table #nome da tabela#;

Informações

describe

o describe 'descreve' uma tabela, ao usar o describe, ele vai mostrar as informações da ESTRUTURA da tabela(não dos registros), ou seja, dos campos

o describe mostra o nome, tipo, se pode ser nulo ou não, a chave, o padrão e algumas Informações extras de cada campo da tabela

é bem simples a sintaxe, olhe:

describe #nome da tabela#;

Select

agora vamos falar do famoso select, o maior comando do MySQL

o uso mais simples e mais usado do select é o select * from(que significa "selecionar tudo de") mostra todos os registros, ou seja, DADOS de uma tabela

essa sintaxe também é bem simples, observe:

select * from #nome da tabela#;

Mas você pode específicar e filtrar mais que dados você quer selecionar, é aí que o comando fica enorme

Filtrando Colunas

Para filtrar colunas, você substitui o asterisco(*) pelos campos que você quer selecionar, e a ordem que você colocar os campos no comando é a ordem que os campos vão aparecer na tela

a sintaxe fica:

select #colunas desejadas# from #nome da tabela#

Você também pode ordenar os resultados baseados em um campo com o order by, se o campo for literal a ordem vai ser alfabética, se for numeral a ordem vai ser ascendente(do menor ao maior), a sintaxe:

select #colunas desejadas# from #nome da tabela# order by #campo referente da ordem#

se quiser que a ordem seja decrecente, é só colocar a palavra desc depois do campo de referência

e outra coisa, é possivel ordenar por um campo, e ainda ordenar por um segundo campo, sendo a prioridade do primeiro, para isso é só colocar os campos separados por vírgulas em ordem, ou seja, o mais importante primeiro e o menos importante depois

por exemplo, se você escolhe os campos de referência como nome e ano, primeiro vai ser ordenado por nome, daí os que tiverem o mesmo nome vão ser ordenados por ano

Filtrar Linhas

Já para filtrar linhas é preciso um pouco mais de trabalho, para isso nós vamos usar o operador já visto anteriormente where,

show databases e show tables

esses dois comandos são usados quando você está trabalhando com os seus bancos de dados no MySQL Console, não no Workbench

eles servem para ver os bancos de dados que seu servidor tem ou as tabelas que o banco de dados selecionado tem

para usa-los é simples, se você usar o seguinte comando vai aparecer uma tabelinha com os bancos de dados do servidor:

show databases;

e com o seguinte comando você verá todas as tabelas do banco de dados selecionado(precisa ter um banco de dados selecionado pelo comando 'use'):

show tables;

e lembrando mais uma vez, esses dois comandos só funcionam no MySQL Console, todos os outros comandos que já vimos podem ser usados tanto no workbench quanto no Console, esses só podem no console, então para mexer no console você pode usar os mesmos comandos do workbench, mas você terá que usar esses dois comandos para ver os banco de dados e tabelas porque não aparece à esquerda da tela que nem no Workbench

Deletando banco de dados e tabelas

drop database

o drop database(de classificação DDL) deleta um banco de dados da sua escolha, a sintaxe é bem simples, olhe:

drop database #nome do banco#;

mas tome cuidado, o drop database é um caminho sem volta, depois de deletar um banco de dados você não pode recuperar ele

drop table

funciona da mesma forma que o drop database, mas com tabelas. Ele também é da classificação DDL

olha que simples a sintaxe:

drop table #nome da tabela#;

você pode usar o if exists no drop table e no drop database para a tabela ou banco de dados só ser deletado se existir, confira isso na página de constraints
