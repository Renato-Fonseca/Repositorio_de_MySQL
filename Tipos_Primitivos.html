Tipos primitivos

Os tipos primitivos são os tipos de dados, você usa tipos primitivos quando você cria ou muda um campo de uma tabela para dizer qual é o tipo de dado que vai ser inserido nele por exemplo

  • Numérico
    • Inteiro
      • tinyint
      • smallint
      • mediumint
      • int
      • bigint
    • Real
      • decimal
      • float
      • double
      • real
    • Lógico
      • bit
      • boolean
  • Data/Tempo
      • date
      • datetime
      • timestamp
      • time
      • year
  • Literal
    • Caractere
      • char
      • varchar
    • Texto
      • tinytext
      • text
      • mediumtext
      • longtext
    • Binário
      • tinyblob
      • blob
      • mediumblob
      • longblob
    • Coleção
      • enum
      • set
  • Espacial
      • geometry
      • point
      • poligon
      • multipoligon

lembrando que o que você vai usar para criar ou modificar campos são os tipos marcados com quadrados pretos, você não vai criar um campo colocando tipo "numérico" por exemplo, mas você pode criar um campo com o tipo "tinyint"

Inteiro+

Se você usar a constraint "unsigned" em um campo com qualquer um das subdivisões do tipo inteiro(ou seja, tinyint, smallint, mediumint, int e bigint), ele não vai aceitar valores negativos, e o alcance dos valores positivos vai aumentar

ou seja, se você cria um campo com o tipo tinyint e com a constraint unsigned, ele não vai aceitar valores negativos, mas agora ele vai aceitar valores positivos até 255, veja só como fica o alcance positivo de cada uma das subdivisões inteiras com a essa constraint:

  • tinyint: 255
  • smallint: 65535
  • mediumint: 16777215
  • int: 4294967295
  • bigint: 18446744073709551615

outra coisa, você pode limitar o alcance maximo dessas subdivisões colocando um parentêses com esse número, é claro que esse número não pode ultrapassar o limite normal

você não pode colocar um limite maximo de 300 em um tinyint por exemplo, mas você pode colocar 60, daí valores maiores que 60 não seram aceitos

Caractere+

A diferença dos tipos char e varchar é que o char é fixo e o varchar é váriavel

No char, você escolhe quantos caracteres serão inseridos, se for inserido menos caracteres do que devia o restante vai ser preenchido com espaços, ou seja, ele sempre vai consumir sempre o mesmo tanto de memória, mesmo sem ser preciso

No varchar você escolhe o máximo de caracteres que vão ser inseridos, mas se for inserido menos, ele vai excluir os espaços não usados, economizando muita memória

Texto+

A diferença de caracteres de textos é que caracteres são um nome ou algo de poucas palavras, já um texto é usado em descrições e textos longos, não confunda

Real+

A diferença dos tipos float, double e decimal são a precisão dos dados

O float e o double guardam qualquer número real, não é fixo, a diferença deles é quanta memória eles consomem

o float consome 4 bytes e tem um limite de casas decimais menor, já o double consome 8 bytes e tem um limite de casa decimais maior

o decimal é mais específico, e o mais útil na maioria dos casos, com ele você escolhe quantas casas o número vai ter no total, e quantas casas decimais ele vai ter

para usar o decimal você deve colocar um parentêses à direita dele, no parentêses você vai colocar dois número separados por uma vírgula, o primeiro é o total de casas no total, e o segundo é quantas casas decimais o número vai ter, para armazenar o número 128.35 por exemplo você precisa usar um decimal(5,2)

se for armazenado um número com mais casas decimais do que o predefinido, o número vai ser arredondado

Coleção+

O enum e o set são muito parecidos, eles servem para limitar as possibilidades de dados que podem ser guardados naquele campo, ou seja, quando você usa enum ou set você pode escolher quais palavras são válidas para aquele campo

A única diferença é que em um campo enum você pode armazenar apenas um dos valores válidos, já no set você pode armazenar quantas opções você quiser

Para usar eles é fácil, você escreve um parênteses à direita deles com os valores permitidos, cada um entre aspas e separados por vírgulas, no enum apenas um desses valores será aceito, no set um ou mais desses valores será aceito
